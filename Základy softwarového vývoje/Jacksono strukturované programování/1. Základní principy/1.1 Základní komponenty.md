### 1. Primitivní operace (Bloky)

Základní, nedekomponovatelná instrukce nebo úloha.

* **Příklad:** Triviální kód pro výpočet ceny s DPH:
   ```javascript
    price = 100;       // cena bez DPH
    vat = 0.21;        // sazba DPH v %
    priceIncludingVat = price * (1 + vat); // cena s DPH
    ```
* Program P je dekomponován na podprogramy P1, P2, P3, dokud nejsou na úrovni **elementárních komponent**, které už dál nemění smysl operace.

### 2. Čtyři základní komponenty JSP

JSP definuje čtyři základní typy komponent, pomocí kterých je možné strukturovat a dekomponovat jak **úlohu (program)**, tak i **data**:

| JSP Komponenta | Popis pro Úlohu/Program | Popis pro Data |
| :--- | :--- | :--- |
| **Sekvence** | Dvě nebo více instrukcí, které se provádějí **v přesném pořadí za sebou**. | Dvě nebo více datových částí, které se vyskytují v přesném pořadí za sebou. |
| **Iterace (cyklus)** | Jedna instrukce, která se **opakuje 0 nebo N-krát** (cyklus). | Jedna datová část, která se vyskytuje **0 nebo N-krát**. |
| **Selekce (výběr)** | Dvě nebo více instrukcí, z nichž se **v daný okamžik provede pouze jedna** (podmíněné větvení). | Dvě nebo více datových částí, z nichž **pouze jedna je přítomna**. |
| **Základní komponenta** | Nejmenší část, která se již dál nedekomponuje. | Nejmenší datová část, která se již dál nedekomponuje (atomická hodnota). |

**K. Dudman** interpretuje tyto komponenty v případě **struktury programu** následovně:

* **Sekvence:** Jedna instrukce za druhou.
* **Iterace:** Opakující se instrukce (cyklus).
* **Selekce:** Jedna instrukce nebo jiná (výběr).

### 3. Kontext a opomenuté informace (Doplnění)

* **Jackson Structured Programming (JSP)** je metodika pro **strukturovaný návrh programů**, kde **struktura programu je odvozena přímo ze struktury dat, které program zpracovává**.
* Základní princip je, že pro efektivní programování musí být **struktura programu izomorfní (shodná)** se strukturou vstupních a výstupních dat.
* JSP používá **diagramy** (známé jako **Jacksonovy diagramy** nebo **strukturované diagramy**) k vizualizaci těchto struktur. Jednotlivé komponenty se graficky znázorňují:
    * **Sekvence:** Obdélníky vedle sebe.
    * **Iterace:** Hvězdička `( * )` v pravém horním rohu obdélníku.
    * **Selekce:** Malé kolečko `(o)` v pravém horním rohu obdélníku.
* **Problém Inkompatibility Struktur (Structure Clash):** JSP se zaměřuje i na situace, kdy struktura vstupních dat není kompatibilní se strukturou výstupních dat. Tento problém se řeší zavedením **intermediárních (přechodných) datových struktur** a rozdělením programu na více sekvenčně spojených částí.
